The plan is complete. Here's a quick orientation of the key decisions:

**Three.js over Canvas 2D** — The PRD explicitly calls for 3D, and Three.js via CDN keeps everything in a single file. All geometry is primitives (spheres, boxes, cylinders, cones) — no assets, no loading.

**Phase Machine as the spine** — `IDLE → CASTING → DESCENDING → PAUSED → ASCENDING → SURFACING → SUMMARY → IDLE` drives everything. Every module only activates in its relevant phases, which keeps the complexity manageable.

**DOM over canvas for UI** — Coin counters, gauges, upgrade buttons, and popups are all DOM elements layered over the canvas. This makes the slot-machine counter animation, coin-fly-to-counter, and upgrade button states trivial to implement correctly with CSS.

**Web Audio API for all sound** — Zero audio files. All SFX are synthesized with `OscillatorNode` and `AudioBufferSourceNode` (white noise). This keeps the file truly self-contained.

**Overspawn fish** — Spawning ~2.5× `maxFish` fish ensures the player always has enough targets to fill their hook, even with imperfect aiming. Only the first `maxFish` collisions register.
